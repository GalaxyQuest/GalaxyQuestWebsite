Chapter 3: Building our 1st Machine and Test Flight

Our heroes set out taking their first pass at training a machine learning algorithm


  • Iteration Based Implementation

    • Our heroes decided on an iterative (build and fix) model for implementing their first machine learning algorithm. With this model, our heroes hoped to take advantage of quickly gaining a high-level view of our project and gaining more insight on priorities. They were guided by a forum posting and set off on their journey.

  • First Features

    • Our heroes viewed the first features as a broad and wholistic set of data. They cropped and grayscaled the images, using the output as an initial set of features. This minimal level of complexity allowed for the fewest complications when they had to pass this data to their machine.

  • Using your Data with your Machine

    • While our heroes were working with images, their machine would not be able to work with raw image data. As a result, our heroes sought additional tools to convert the images into a readable format. They saved these new inputs into CSV files, making sure the headers and labels were properly aligned. They were off to train their machine.

  • Interpreting your Output

    • A set of training images and answers had been set aside for testing from the beginning. After the machine had been trained, they were able to run predictions and test their machine's accuracy with real data. They discovered that despite their small sample size, they were still 84% accurate. From here, our heroes were able to predict different aspects of our solution in order to get more granular data for testing.

Lessons Learned and Discovered Best Practices





  • Machine Learning Implementation

    • There are two theories behind implementing machine learning algorithms. The first theory requires careful planning and can produce a more scalable and elegant algorithm. The danger of this approach is that it can lead to misprioritization of work, as the task ahead is still relatively hazy. Our strategy was a "build-and-fix" strategy. In this implementation, the goal is to implement an algorithm quickly and use extensive testing to identify areas to improve. The advantage of this approach is that more time is spent on relevant tasks.

  • First Features

    • In the first feature extraction, the goal is to use a feature set that can provide a good approximation for the question you ask. Common image features are listed here. Among other data sets, common features may look like a dictionary of related words or common attributes.

  • Using your Data with your Machine

    • Your machine will require data input to be in a consistent style. Headers, footers and labels are common formatting decisions that should be set as standards for your project.   • Interpreting your Output

    • When interpreting your output, it is useful to measure your success and also be able to assess where your algorithm is weak. We developed error testing algorithms to get a more granular look at our data. We discovered that, our algorithm was weakest when classifying sphere and spiral galaxies, but very precise in identifying galactic anomalies. We were able to use these granular outputs to further guide future iterations. This guide on error testing was a great resource for us.
